## 🪄 프로젝트 소개

> **개발자를 위한 지식을 수집하고 연결하는 AI 지식 관리 서비스**

개발을 하다 보면 GitHub, 문서, 블로그, 메모 등 다양한 곳에 정보가 쌓이지만,  
정작 필요한 순간에 다시 꺼내 보고 연결하기는 어렵습니다.

이 프로젝트는 흩어진 개발 지식을 빠르게 수집하고,  
현재 맥락에 맞게 다시 연결해 더 쉽게 활용할 수 있도록 돕는 서비스를 만드는 것을 목표로 합니다.

## 🪄 기획 배경

개발 과정에서는 많은 정보와 기록이 쌓이지만,  
대부분은 저장된 뒤 다시 활용되지 못한 채 흩어져 남게 됩니다.

기존에 봤던 자료나 학습한 내용이 현재 작업과 자연스럽게 연결되지 않다 보니,  
비슷한 내용을 반복해서 찾거나 다시 정리해야 하는 경우가 자주 발생합니다.

저희는 이런 문제를 줄이기 위해,  
지식을 **수집하고, 정리하고, 다시 연결하는 흐름** 자체를 더 자연스럽게 만들고자 합니다.

---

## 🪄 핵심 기능 (MVP 기준)

### 1. 원탭 지식 수집

> **빠른 입력, 낮은 저장 장벽**

- 드래그와 단축키를 활용해 필요한 정보를 빠르게 수집
- 캡처, 링크 저장 등 다양한 방식으로 지식을 기록
- 사용자 액션 기반으로 수집해 불필요한 노이즈를 줄이고, 필요한 정보 중심으로 정리

### 2. GitHub 기반 초기 컨텍스트 확보

> **개발 흐름 반영, 콜드 스타트 완화**

- GitHub 레포지토리를 연동해 커밋, README 등 개발 흔적을 기반으로 초기 맥락 구성
- 별도 입력이 많지 않은 초기 단계에서도 사용자의 학습 흐름을 반영할 수 있도록 지원
- 서비스에서 정리된 학습 내용을 Markdown으로 자동 정리하고, GitHub 레포에 학습 기록 형태로 축적할 수 있도록 설계
- 수집된 GitHub 정보는 이후 추천, 요약, 리콜의 기초 데이터로 활용

### 3. 실시간 컨텍스트 기반 추천

> **현재 맥락에 맞는 지식 연결**

- 사용자가 현재 보고 있는 내용을 기준으로 관련된 과거 지식을 자동 연결
- 관련 기록이 있을 경우 별도 검색 없이 바로 다시 확인 가능
- 연결할 데이터가 부족한 경우에는 핵심 요약과 확장 학습 키워드 제공

### 4. AI 지식 카드 생성

> **비정형 정보의 구조화**

- 수집된 원본 데이터를 카드 형태로 정리해 빠르게 파악할 수 있도록 구성
- 자동 태깅, 3줄 요약, 헤더 생성 등을 통해 핵심 내용을 구조화
- 메타데이터를 함께 저장해 이후 검색, 연결, 리콜에 활용 가능하도록 설계

### 5. 하루 1회 학습 리콜

> **복습 자동화와 학습 흐름 유지**

- 전날 학습한 내용을 요약해 다시 제공
- 과거 데이터와 연결해 흐름이 이어지는 복습 경험 지원
- 추가 학습 방향을 제안해 다음 학습으로 자연스럽게 이어질 수 있도록 구성
